# Future Improvements

## Purpose

This document tracks realistic, non-promissory ideas for the NOVA BLOCKS repository. Items here are suggestions for future work, not commitments or verified delivery dates.

## Proposed Areas

### Frontend

- Improve component organization and reuse.
- Add clearer loading, error, and empty states.
- Expand automated UI testing coverage.

### Backend

- Add more route-level and integration tests.
- Improve API validation and error responses.
- Document runtime configuration and environment variables.

### Finance and Benchmarking

- Separate experimental scripts from maintained utilities.
- Add reproducible test data for benchmark runs.
- Document expected inputs, outputs, and failure modes.

### Documentation

- Keep planning documents clearly labeled as aspirational where appropriate.
- Reduce duplication across revenue and ownership notes.
- Add a short architecture index that points to authoritative source files.

## Guidance

Any future roadmap item should include:

- the problem being solved
- the current implementation status
- the owner or maintainer
- the validation method

## Status

Aspirational and informational only.
